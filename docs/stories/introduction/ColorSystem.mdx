# 색상 시스템

루미르 디자인 시스템의 색상 시스템은 일관된 시각적 경험을 제공하기 위한 체계적인 색상 구조를 정의합니다. 색상 토큰은 Foundation과 Semantic 레이어로 구성되어 있습니다.

## 색상 원칙

1. **일관성**: 모든 제품과 플랫폼에서 일관된 색상 경험 제공
2. **접근성**: WCAG 2.1 AA 기준을 준수하는 대비율 유지
3. **의미 전달**: 색상을 통해 명확한 의미와 상태 전달
4. **확장성**: 다양한 제품과 테마에 적용 가능한 유연한 구조

## 색상 사용 가이드라인

### 1. 색상 토큰 사용

* 직접적인 색상 값(HEX, RGB) 대신 항상 Semantic 색상 토큰을 사용하세요.
* CSS에서는 `var(--semantic-color-primary-background-1-rest)` 형식으로 사용합니다.

```css
/* 권장 */
.button {
  background-color: var(--semantic-color-primary-background-1-rest);
  color: var(--semantic-color-primary-oncolor-global);
}

/* 권장하지 않음 */
.button {
  background-color: #2196F3;
  color: #FFFFFF;
}
```

### 2. 접근성

* 텍스트와 배경 색상 간의 대비율이 WCAG 2.1 AA 기준을 충족하는지 확인하세요.
  * 일반 텍스트(16px 이하): 최소 4.5:1 대비율
  * 큰 텍스트(18px 이상): 최소 3:1 대비율
* 색상만으로 정보를 전달하지 마세요. 항상 추가적인 시각적 요소나 텍스트를 함께 사용하세요.

### 3. 색상 조합

* Primary와 Secondary 색상을 적절히 조합하여 시각적 계층 구조를 형성하세요.
* 한 화면에 너무 많은 강조 색상을 사용하지 마세요. 중요한 액션과 정보에만 강조 색상을 사용하세요.
* 상태 색상(오류, 경고, 성공, 정보)은 일관되게 사용하여 사용자가 쉽게 의미를 파악할 수 있게 하세요.

## 다크 모드

루미르 디자인 시스템은 라이트 모드와 다크 모드를 모두 지원합니다. 다크 모드에서는 다음 원칙을 적용합니다:

* 눈의 피로를 줄이기 위해 대비를 낮춥니다.
* 어두운 배경에서 더 잘 보이도록 강조 색상의 채도를 높입니다.
* 모든 텍스트는 WCAG 접근성 기준을 계속 충족해야 합니다.

다크 모드 전환은 Semantic 토큰 레이어에서 처리되므로, 컴포넌트 코드를 변경할 필요 없이 테마가 자동으로 적용됩니다.
