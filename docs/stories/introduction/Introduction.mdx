# 루미르 디자인 시스템 (LDS)

루미르 디자인 시스템은 일관된 사용자 경험을 제공하며 효율적인 디자인 및 개발 워크플로우를 지원하는 디자인 가이드라인, 컴포넌트 및 패턴의 통합 시스템입니다.

## 디자인 시스템 소개

루미르 디자인 시스템은 다양한 서비스와 플랫폼에서 일관성 있고 효율적인 사용자 경험을 제공하기 위해 설계되었습니다. 
디자인 시스템의 핵심 요소는 다음과 같습니다:

### 토큰 기반 시스템

- **파운데이션 토큰**: 기본적인 디자인 값(색상 코드, 크기, 간격 등)을 정의
- **시맨틱 토큰**: 실제 사용 목적과 맥락에 따라 파운데이션 토큰에 의미를 부여

### 컴포넌트 계층 구조

- **Primitives**: 기본 UI 요소 (버튼, 입력 필드, 아이콘 등)
- **Compounds**: 여러 기본 요소로 구성된 복합 UI 컴포넌트
- **Patterns**: 일반적인 UI 패턴 및 레이아웃 구조

## 사용 방법

### 설치

```bash
npm install @lumir/design-system
```

### 기본 사용법

```jsx
import { Button, TextField } from '@lumir/design-system';

function MyComponent() {
  return (
    <div>
      <TextField label="이름" placeholder="이름을 입력하세요" />
      <Button variant="primary">제출</Button>
    </div>
  );
}
```

### 스타일 적용

CSS 변수를 통해 디자인 토큰을 쉽게 적용할 수 있습니다:

```css
.my-element {
  color: var(--semantic-color-primary-foreground-1-rest);
  padding: var(--semantic-spacingHor-global-md);
  border-radius: var(--semantic-radius-global-sm);
}
```

## 디자인 원칙

루미르 디자인 시스템은 다음 원칙을 바탕으로 구축되었습니다:

1. **일관성**: 모든 제품과 플랫폼에서 일관된 사용자 경험 제공
2. **접근성**: 포괄적이고 접근 가능한 디자인으로 모든 사용자 포용
3. **확장성**: 다양한 제품과 요구사항에 맞게 확장 가능한 시스템
4. **효율성**: 디자인과 개발 과정의 효율성 향상
5. **유연성**: 다양한 브랜드와 맥락에 적용 가능한 유연한 시스템

## 디자인 시스템 구조

```
project/
├── design-systems/          # 디자인 시스템 정의
│   ├── system-1/            # 디자인 시스템 1 (서비스 A용)
│   │   ├── tokens/          # 토큰 정의
│   │   │   ├── semantic.json
│   │   ├── components/      # 컴포넌트 라이브러리
│   │   │   ├── primitives/  # 기본 UI 요소
│   │   │   ├── compounds/   # 복합 UI 요소
│   │   │   └── patterns/    # 패턴
│   │   ├── styles/          # 생성된 CSS 변수
│   │   │   └── variables.css
├── shared/                  # 공유 리소스
│   └── tokens/              # 공유 토큰
│       └── foundation.json  # 기초 토큰
```
