# 디자인 패턴

디자인 패턴은 루미르 디자인 시스템에서 일관된 사용자 경험을 제공하기 위한 표준화된 UI 패턴과 상호작용 방식을 정의합니다. 이러한 패턴은 사용자가 제품을 보다 쉽게 이해하고 사용할 수 있도록 돕습니다.

## 주요 디자인 패턴

### 1. 폼 패턴

폼은 사용자로부터 정보를 수집하는 중요한 인터페이스 요소입니다. 루미르 디자인 시스템은 일관된 폼 디자인과 상호작용 방식을 제공합니다.

#### 폼 레이아웃

- **수직 레이아웃**: 기본적으로 폼 요소는 수직으로 배치
- **그룹화**: 관련 입력 필드는 논리적으로 그룹화
- **적절한 간격**: 요소 간 일관된 간격 유지

```jsx
<Form>
  <FormGroup label="개인 정보">
    <TextField label="이름" required />
    <TextField label="이메일" type="email" required />
  </FormGroup>
  
  <FormGroup label="배송 정보">
    <TextField label="주소" />
    <TextField label="우편번호" />
  </FormGroup>
  
  <ButtonGroup>
    <Button variant="secondary">취소</Button>
    <Button variant="primary">제출</Button>
  </ButtonGroup>
</Form>
```

#### 유효성 검증

- **인라인 검증**: 사용자가 필드를 벗어날 때 즉시 피드백 제공
- **제출 시 검증**: 폼 제출 시 모든 필드 검증
- **명확한 오류 메시지**: 문제 해결 방법을 안내하는 구체적인 메시지

### 2. 내비게이션 패턴

내비게이션은 사용자가 제품 내에서 쉽게 이동할 수 있도록 돕는 패턴입니다.

#### 글로벌 내비게이션

- **일관된 위치**: 모든 화면에서 동일한 위치에 배치
- **현재 위치 표시**: 사용자의 현재 위치를 명확히 표시
- **계층 구조**: 주요 섹션과 하위 섹션을 구분

#### 페이지 내 내비게이션

- **탭**: 동일한 수준의 콘텐츠를 구분하는 데 사용
- **사이드 내비게이션**: 깊은 계층 구조를 표현하는 데 적합
- **브레드크럼**: 사용자의 현재 위치와 탐색 경로 표시

### 3. 데이터 표시 패턴

데이터를 효과적으로 표시하는 패턴으로, 사용자가 정보를 쉽게 이해하고 분석할 수 있도록 도와줍니다.

#### 테이블

- **정렬**: 열 헤더를 클릭하여 정렬 기능 제공
- **필터링**: 표시할 데이터를 필터링하는 옵션 제공
- **페이지네이션**: 대량의 데이터를 관리 가능한 페이지로 분할

```jsx
<DataTable
  columns={[
    { key: 'name', header: '이름', sortable: true },
    { key: 'email', header: '이메일' },
    { key: 'role', header: '역할', filterable: true }
  ]}
  data={userData}
  pagination={{ pageSize: 10 }}
/>
```

#### 카드

- **일관된 레이아웃**: 모든 카드가 동일한 구조 유지
- **계층 구조**: 중요한 정보를 먼저 배치
- **상호작용**: 클릭, 호버 등의 상호작용 패턴 일관되게 적용

### 4. 피드백 패턴

사용자 행동에 대한 피드백을 제공하는 패턴으로, 사용자가 시스템의 상태를 이해하는 데 도움을 줍니다.

#### 알림

- **토스트**: 일시적인 피드백에 사용 (예: 저장 성공)
- **모달**: 중요한 정보나 확인이 필요한 경우 사용
- **인라인 알림**: 특정 섹션이나 컨텍스트 내 피드백 제공

#### 로딩 상태

- **스켈레톤 로딩**: 콘텐츠 구조를 미리 보여주는 로딩 상태
- **진행 표시기**: 긴 작업의 진행 상황 표시
- **미니멀 로딩**: 작은 액션에 대한 간결한 로딩 표시

## 패턴 적용 원칙

### 1. 일관성

- 동일한 상황에서는 항상 동일한 패턴 사용
- 플랫폼 간(웹, 모바일, 태블릿 등) 일관된 경험 제공

### 2. 단순성

- 불필요한 복잡성 제거
- 사용자의 목표 달성을 위한 최단 경로 제공

### 3. 접근성

- 다양한 사용자와 상황을 고려한 디자인
- 모든 상호작용은 키보드로도 가능하게 설계

### 4. 사용자 피드백 반영

- 사용자 테스트 결과를 반영하여 패턴 지속적 개선
- 실제 사용 데이터를 기반으로 패턴 최적화

## 패턴 확장 및 사용자 정의

디자인 패턴은 기본 구조와 원칙을 제공하지만, 특정 제품이나 상황에 맞게 확장하거나 사용자 정의할 수 있습니다. 주요 패턴을 수정할 때는 다음 원칙을 따릅니다:

- 핵심 상호작용 방식 유지
- 일관된 시각적 언어 유지
- 변경 사항에 대한 충분한 근거와 문서화
- 접근성 표준 준수
