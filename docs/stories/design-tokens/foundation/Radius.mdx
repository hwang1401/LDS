## 반경 선택 가이드라인

UI 요소의 크기와 중요도에 따라 적절한 반경을 선택하는 것이 중요합니다:

- **작은 요소 (아이콘, 배지)**: radius-20 ~ radius-40 (2px ~ 4px)
- **중간 요소 (버튼, 입력 필드)**: radius-40 ~ radius-80 (4px ~ 8px)
- **큰 요소 (카드, 모달)**: radius-80 ~ radius-160 (8px ~ 16px)
- **원형 요소**: radius-1000 (10000px 또는 50%)

## 사용 가이드라인

Foundation 반경 토큰은 직접 사용보다는 Semantic 토큰을 통해 사용하는 것이 권장됩니다. 이는 디자인 변경 시 일관성을 유지하고 유지보수를 용이하게 합니다.

```css
/* 권장하지 않음 */
.button {
  border-radius: var(--foundation-radius-40);
}

/* 권장 */
.button {
  border-radius: var(--semantic-radius-global-sm);
}
```

일관된 반경 시스템을 사용하면 다음과 같은 이점이 있습니다:

- **시각적 일관성**: 모든 UI 요소에 일관된 모서리 처리 제공
- **브랜드 아이덴티티**: 특정 반경을 통해 브랜드의 시각적 특성 강화
- **사용자 경험**: 적절한 반경을 통해 요소의 상호작용 가능성과 우선순위 전달
